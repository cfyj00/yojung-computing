| 항목 | 내용 |

| **소요 시간** | 약 30분 |
| **완료 기준** | `print("Hello")` 를 실행하여 결과를 확인한다 |

### 개념 설명: 파이썬이란?

> **💬 AI 선생님 스크립트**
>
> 오늘 첫 강의입니다. 환영합니다.
>
> 파이썬이 뭔지 한 문장으로 설명하면: **"컴퓨터에게 지시를 내리는 언어"**입니다. 우리가 한국어로 사람에게 말하듯, 파이썬으로 컴퓨터에게 말합니다.
>
> 예를 들어볼게요. 카페에서 직원에게 이렇게 말합니다:
> "오늘 아메리카노 판매량을 세어서 알려줘"
>
> 파이썬으로는 이렇게 말합니다:
> `print(아메리카노_판매량)`
>
> `print()`는 "화면에 출력해줘"라는 뜻입니다. 파이썬의 가장 첫 번째 명령어입니다.

### 첫 번째 코드

```python
# #(샵)으로 시작하는 줄은 주석입니다
# 주석 = 컴퓨터는 무시하고, 사람만 읽는 설명
# 코드가 뭘 하는지 메모해두는 공간입니다

# print() = "이것을 화면에 출력해줘"라는 명령어
print("안녕하세요, 요정코드 학원입니다!")

# 숫자도 출력할 수 있습니다
print(42)

# 계산 결과도 출력할 수 있습니다
print(10 + 5)
```

**📤 예상 출력 결과**
```
안녕하세요, 요정코드 학원입니다!
42
15
```

> **⚠️ 자주 나오는 첫 번째 오류**
>
> `SyntaxError: EOL while scanning string literal`
> - 원인: 따옴표를 열었는데 닫지 않음
> - 예: `print("안녕하세요)` ← 닫는 따옴표 없음
> - 해결: `print("안녕하세요")` ← 따옴표 짝을 맞춤
>
> `NameError: name ... is not defined`
> - 원인: 파이썬이 모르는 이름을 사용함
> - 해결: 따옴표로 감싸거나 (문자열로 만들거나) 변수를 먼저 정의

## 강의 1.2 — 변수: 정보를 담는 상자

| 항목 | 내용 |
|------|------|
| **목표** | 변수가 무엇인지 이해하고 다양한 타입의 변수를 만들 수 있다 |
| **소요 시간** | 약 40분 |
| **완료 기준** | 변수 5개를 만들고 각각 다른 타입임을 `type()`으로 확인한다 |

### 개념: 변수란?

> **📦 비유: 변수 = 이름표 붙은 상자**
>
> 카페 창고를 생각해보세요. 원두 상자에 "에티오피아 원두"라고 이름표를 붙입니다. 나중에 "에티오피아 원두"라고 부르면 그 상자를 찾아옵니다.
>
> 변수도 같습니다:
> ```
> 원두 = "에티오피아 예가체프"
>  ↑           ↑
> 이름표    상자 안의 내용
> ```
>
> 나중에 "원두"라고 부르면 "에티오피아 예가체프"가 나옵니다.
>
> **변수를 쓰는 이유**: 같은 값을 여러 번 쓸 때 변수 이름만 바꾸면 됨

> **💬 AI 선생님 스크립트**
>
> 카페에서 아메리카노 가격이 4,500원입니다. 영수증, 메뉴판, 키오스크에 모두 4500을 적었습니다. 어느 날 가격이 4,800원으로 오릅니다.
>
> - 변수를 **안 쓰면**: 4500이 적힌 모든 곳을 찾아서 하나하나 바꿔야 합니다.
> - 변수를 **쓰면**: `아메리카노_가격 = 4800` — 딱 한 줄만 바꾸면 됩니다.

### 변수 만들기

```python
# ── 변수 만드는 방법 ─────────────────────────────
# 변수이름 = 값
# = (등호)는 "오른쪽 값을 왼쪽 이름에 저장해줘"라는 뜻

# 문자열(str): 글자를 담는 변수 — 따옴표로 감쌈
카페명 = "요정카페"
대표메뉴 = "콜드브루"

# 정수(int): 소수점 없는 숫자
아메리카노_가격 = 4500
오늘_판매량 = 87

# 실수(float): 소수점 있는 숫자
원두_잔량_kg = 2.5
부가세율 = 0.1

# 불리언(bool): 참(True) 또는 거짓(False)만 담는 변수
카페_영업중 = True
재고_부족 = False

# 변수에 담긴 값 출력하기
print(카페명)
print(아메리카노_가격)
print(원두_잔량_kg)
print(카페_영업중)
```

**📤 예상 출력 결과**
```
요정카페
4500
2.5
True
```

### 타입 확인하기

```python
# type() = "이 변수가 어떤 타입인지 알려줘"
print(type(카페명))           # 출력: <class 'str'>
print(type(아메리카노_가격))   # 출력: <class 'int'>
print(type(원두_잔량_kg))     # 출력: <class 'float'>
print(type(카페_영업중))      # 출력: <class 'bool'>

# 타입이 왜 중요한가?
# 숫자끼리는 더할 수 있지만, 문자와 숫자는 더할 수 없습니다
# 잘못된 타입끼리 연산하면 오류가 납니다

# 올바른 예시
오늘_매출 = 아메리카노_가격 * 오늘_판매량
print(오늘_매출)  # 출력: 391500

# 틀린 예시 — 실행하면 오류 납니다
# print(카페명 + 아메리카노_가격)  # 문자 + 숫자 = 오류!
```

### 변수 이름 짓는 규칙

| **불가능한 것** | 공백 불가, 특수문자 불가(@, !, #), 숫자로 시작 불가 |

### f-string: 변수를 문장에 끼워넣기

```python
# f-string = 변수를 문장 중간에 자연스럽게 넣는 방법
# f"..." 형식으로 쓰고, 변수는 {} 안에 넣습니다

카페명 = "요정카페"
오늘_판매량 = 87
아메리카노_가격 = 4500

# f-string 없이 — 불편함
print("오늘 " + 카페명 + "의 판매량은 " + str(오늘_판매량) + "잔입니다")

# f-string 사용 — 훨씬 깔끔함
print(f"오늘 {카페명}의 판매량은 {오늘_판매량}잔입니다")

# 계산 결과도 바로 넣을 수 있습니다
오늘_매출 = 오늘_판매량 * 아메리카노_가격
print(f"오늘 매출: {오늘_매출:,}원")  # :, 는 1,000 단위 콤마 표시
```

**📤 예상 출력 결과**
```
오늘 요정카페의 판매량은 87잔입니다
오늘 매출: 391,500원
```

## 강의 1.3 — 조건문: 상황에 따라 다르게 행동하기

| 항목 | 내용 |
|------|------|
| **목표** | if/elif/else로 조건에 따라 다른 코드를 실행할 수 있다 |
| **소요 시간** | 약 40분 |
| **완료 기준** | 3가지 조건이 있는 if-elif-else 구조를 직접 작성할 수 있다 |

> **🔀 비유: 조건문 = 카페 주문 처리 규칙**
>
> 카페에서 음료를 만들 때 규칙이 있습니다:
> - 만약(if) 고객이 아이스를 원하면 → 얼음을 넣어라
> - 아니면 만약(elif) 고객이 뜨겁게 원하면 → 뜨거운 물을 넣어라
> - 그 외(else) → "아이스인지 핫인지 확인해주세요"라고 말해라
>
> 조건문은 "상황에 따라 다르게 행동하는 것"입니다.

```python
# ── 조건문 기본 구조 ─────────────────────────────
# if 조건:        ← 조건이 True면 아래 코드 실행
#     실행할 코드  ← 들여쓰기(Tab) 필수!
# elif 다른조건:   ← 첫 조건이 False이고 이 조건이 True면
#     실행할 코드
# else:           ← 위 모든 조건이 False면
#     실행할 코드

# ── 카페 재고 경보 시스템 ────────────────────────
원두_잔량_kg = 1.2  # 현재 원두 재고

if 원두_잔량_kg < 0.5:
    # 들여쓰기(Tab)로 "이 코드는 if 안에 있다"를 표시
    print("🚨 긴급! 원두가 거의 없습니다. 즉시 주문하세요!")
elif 원두_잔량_kg < 2.0:
    print("⚠️ 원두가 부족합니다. 곧 주문이 필요합니다.")
else:
    print("✅ 원두 재고가 충분합니다.")
```

**📤 예상 출력 결과**
```
⚠️ 원두가 부족합니다. 곧 주문이 필요합니다.
```

### 비교 연산자 — 조건을 만드는 방법

| 연산자 | 의미 | 예시 | 해석 |
|--------|------|------|------|
| `==` | 같다 | `원두 == "에티오피아"` | 원두가 에티오피아인가? |
| `!=` | 다르다 | `날씨 != "비"` | 날씨가 비가 아닌가? |
| `>` | 초과 | `판매량 > 100` | 100잔 초과인가? |
| `>=` | 이상 | `재고 >= 10` | 10 이상인가? |
| `<` | 미만 | `재고 < 5` | 5 미만인가? |
| `<=` | 이하 | `가격 <= 5000` | 5000 이하인가? |

### 들여쓰기: 파이썬의 핵심 규칙

> **⚠️ 들여쓰기는 파이썬의 생명입니다**
>
> 파이썬은 들여쓰기(Tab 또는 스페이스 4개)로 "어떤 코드가 안에 있는지" 구분합니다.
>
> 올바른 예:
> ```python
> if 재고 < 10:
>     print("재고 부족")  ← Tab으로 들여써야 합니다
> ```
>
> 틀린 예 (IndentationError 발생):
> ```python
> if 재고 < 10:
> print("재고 부족")  ← 들여쓰기 없으면 오류!
> ```
>
> 해결: if/elif/else/for/while 다음 줄은 항상 Tab으로 들여쓰기

> **📝 실습 과제**
> 1. 로스팅 공장 품질 체크 시스템 만들기: `로스팅_시간` 변수(초 단위), 8~12분(480~720초) 이내면 "정상", 8분 미만이면 "과소 로스팅", 12분 초과면 "과다 로스팅"
> 2. 카페 할인 시스템 만들기: 주문금액 20,000원 이상이면 10% 할인, 15,000원 이상이면 5% 할인, 그 미만이면 할인 없음 → 최종 결제 금액 출력

> **✅ 이해도 확인 질문**
> - "if, elif, else의 차이를 카페 규칙으로 설명해보세요"
> - "들여쓰기가 없으면 어떤 오류가 나나요?" → IndentationError
> - "and와 or의 차이는?" → and: 둘 다 True / or: 하나라도 True

## 강의 1.5 — 함수: 코드를 레고처럼 조립하기

| 항목 | 내용 |
|------|------|
| **목표** | 함수를 만들고 재사용할 수 있다. 매개변수와 반환값을 이해한다 |
| **소요 시간** | 약 50분 |
| **완료 기준** | 입력을 받아 계산하고 결과를 반환하는 함수를 직접 만든다 |

> **🧩 비유: 함수 = 카페 레시피 카드**
>
> 카페에서 아메리카노 만드는 레시피 카드가 있습니다:
> - 레시피명: 아메리카노_만들기
> - 재료(입력): 에스프레소 양, 물 양
> - 과정: 에스프레소 추출 → 물 붓기 → 잘 섞기
> - 결과(출력): 완성된 아메리카노
>
> 레시피 카드를 만들어두면 → 언제든 꺼내서 재사용
> 함수를 만들어두면 → 언제든 호출해서 재사용

```python
# ── 함수 기본 구조 ───────────────────────────────
# def 함수이름(매개변수):  ← def = "define(정의하다)"
#     내부 코드
#     return 반환값      ← 결과를 돌려줌

# ── 예시 1: 매출 계산 함수 ──────────────────────
def 매출_계산(판매량, 가격):
    """
    판매량과 가격을 받아서 총 매출을 계산합니다.
    이 설명(docstring)은 함수가 뭘 하는지 적는 공간입니다.
    """
    # 함수 안에서 계산
    총_매출 = 판매량 * 가격
    # return = "이 값을 돌려줘"
    return 총_매출

# 함수 호출 (사용하기)
아메리카노_매출 = 매출_계산(87, 4500)  # 판매량=87, 가격=4500
print(f"아메리카노 매출: {아메리카노_매출:,}원")

라떼_매출 = 매출_계산(45, 5500)  # 같은 함수, 다른 값
print(f"라떼 매출: {라떼_매출:,}원")
```

**📤 예상 출력 결과**
```
아메리카노 매출: 391,500원
라떼 매출: 247,500원
```

### 주 1 프로젝트 — 카페 하루 매출 계산기

```python
# ── 주 1 미니 프로젝트: 카페 하루 매출 계산기 ────

# 데이터 정의
메뉴_정보 = {
    "아메리카노": {"가격": 4500, "판매량": 87},
    "카페라떼":   {"가격": 5500, "판매량": 45},
    "에스프레소": {"가격": 4000, "판매량": 23},
    "콜드브루":   {"가격": 6000, "판매량": 31},
}

# 함수 정의
def 메뉴_매출(메뉴명, 메뉴_데이터):
    """메뉴 이름과 데이터를 받아 매출 계산"""
    가격 = 메뉴_데이터["가격"]
    판매량 = 메뉴_데이터["판매량"]
    return 가격 * 판매량

def 일일_리포트(메뉴_정보):
    """전체 메뉴 일일 리포트 출력"""
    print("=" * 40)
    print("요정카페 오늘 매출 리포트")
    print("=" * 40)

    총_매출 = 0
    best_menu = None
    best_매출 = 0

    for 메뉴명, 데이터 in 메뉴_정보.items():
        매출 = 메뉴_매출(메뉴명, 데이터)
        총_매출 += 매출
        print(f"{메뉴명:10s}: {데이터['판매량']:3d}잔 × {데이터['가격']:,}원 = {매출:,}원")
        if 매출 > best_매출:
            best_매출 = 매출
            best_menu = 메뉴명

    print("-" * 40)
```

> **🗂️ 비유: 딕셔너리 = 메뉴판**
>
> | 메뉴명(키) | → | 가격(값) |
> |-----------|---|---------|
> | 아메리카노 | → | 4,500원 |
> | 카페라떼 | → | 5,500원 |
>

```python
# ── 딕셔너리 기본 ───────────────────────────────

    "카페라떼": 5500,
    "에스프레소": 4000,
    "콜드브루": 6000
}

# 키로 값 가져오기
print(메뉴_가격["아메리카노"])   # 4500

# .get()으로 안전하게 가져오기 (없으면 None 반환)
print(메뉴_가격.get("녹차라떼", "메뉴없음"))  # 메뉴없음

# 추가/수정
메뉴_가격["카푸치노"] = 5500    # 새 항목 추가
메뉴_가격["아메리카노"] = 4800  # 기존 값 수정

# 삭제
del 메뉴_가격["에스프레소"]     # 키로 삭제

# 순회
for 메뉴명, 가격 in 메뉴_가격.items():
    print(f"{메뉴명}: {가격:,}원")

# 키만, 값만
print(list(메뉴_가격.keys()))   # 키 목록
print(list(메뉴_가격.values())) # 값 목록
```

## 강의 3.1 — 클래스: 나만의 데이터 타입 만들기

| 항목 | 내용 |
|------|------|
| **목표** | 클래스와 인스턴스의 개념을 이해하고 직접 만들 수 있다 |
| **소요 시간** | 약 50분 |
| **완료 기준** | 막걸리 제품 클래스를 만들고 여러 인스턴스를 생성할 수 있다 |

> **🏭 비유: 클래스 = 제품 설계도, 인스턴스 = 실제 제품**
>
> 설계도(클래스): "막걸리 제품이란 이런 것이다" — 이름/도수/가격이 있고, 재고를 조회하고, 출하 가능 여부를 확인할 수 있다
>
> 실제 제품(인스턴스): 요정막걸리_클래식(6도, 4500원), 요정막걸리_스파클링(5도, 6000원)
>
> 설계도 하나로 여러 제품을 만들 수 있습니다.

```python
# ── 클래스 기본 구조 ─────────────────────────────
class 막걸리제품:
    """막걸리 제품을 표현하는 클래스"""

    def __init__(self, 이름, 도수, 가격, 재고=0):
        """
        __init__ = "이 클래스로 제품을 만들 때 실행되는 함수"
        self = "이 제품 자신"을 가리키는 특별한 이름
        """
        # self.이름 = 이 제품의 이름 속성
        self.이름 = 이름
        self.도수 = 도수
        self.가격 = 가격
        self.재고 = 재고
        self.판매량 = 0  # 처음엔 판매 0

    def 판매(self, 수량):
        """제품을 판매하는 메서드"""
        if 수량 > self.재고:
            print(f"재고 부족! 현재 재고: {self.재고}병")
            return False
        self.재고 -= 수량       # 재고 감소
        self.판매량 += 수량     # 판매량 증가
        print(f"{self.이름} {수량}병 판매 완료")
        return True

    def 재고_추가(self, 수량):
        """재고를 추가하는 메서드"""
        self.재고 += 수량
        print(f"{self.이름} 재고 {수량}병 추가. 총 재고: {self.재고}병")

    def 현황(self):
        """현재 상태를 출력하는 메서드"""
        총_매출 = self.판매량 * self.가격
        print(f"--- {self.이름} ---")
        print(f"  도수: {self.도수}도 / 가격: {self.가격:,}원")
        print(f"  재고: {self.재고}병 / 누적판매: {self.판매량}병")
        print(f"  누적매출: {총_매출:,}원")

# ── 클래스로 제품 만들기 (인스턴스 생성) ─────────
클래식 = 막걸리제품("요정막걸리 클래식", 도수=6.0, 가격=4500, 재고=300)
스파클링 = 막걸리제품("요정막걸리 스파클링", 도수=5.0, 가격=6000, 재고=150)

# 판매 처리
클래식.판매(50)
클래식.판매(30)
스파클링.판매(20)

# 재고 추가
클래식.재고_추가(100)

# 현황 확인
클래식.현황()
스파클링.현황()
```

```python
# ── 예외 처리 기본 ──────────────────────────────
# try: 오류가 날 수 있는 코드
# except: 오류가 났을 때 실행할 코드
# finally: 오류 여부와 관계없이 항상 실행

def 파일_읽기_안전(파일명):
    """파일이 없어도 프로그램이 멈추지 않는 안전한 읽기"""
    try:
        with open(파일명, "r", encoding="utf-8") as f:
            return f.read()
    except FileNotFoundError:
        # 파일이 없을 때
        print(f"파일을 찾을 수 없습니다: {파일명}")
        return None
    except Exception as e:
        # 그 외 모든 오류
        print(f"파일 읽기 오류: {e}")
        return None

# 있는 파일
내용 = 파일_읽기_안전("inventory.json")

# 없는 파일 — 오류 없이 처리됨
내용2 = 파일_읽기_안전("없는_파일.json")

            max_tokens=256,
            messages=[{"role": "user", "content": 질문}]
        )
        return response.content[0].text
    except anthropic.RateLimitError:
        print("API 요청이 너무 많습니다. 잠시 후 다시 시도하세요.")
        return None
    except anthropic.AuthenticationError:
        print("API 키가 올바르지 않습니다.")

    except Exception as e:
        print(f"예상치 못한 오류: {e}")
        return None
```

```python
def 원두_원가_계산(원두_가격_달러, 중량_kg):
    """달러 기준 원두 가격을 원화로 환산합니다"""
    try:
        응답 = requests.get("https://api.exchangerate-api.com/v4/latest/USD",
                            timeout=10)  # 10초 이내에 응답 없으면 오류

        kg당_원가 = 원화_가격 / 중량_kg
        return round(kg당_원가)
    except requests.exceptions.Timeout:
        print("API 응답 시간 초과")
        return None

원가 = 원두_원가_계산(8.5, 1)  # 1kg당 8.5달러
```

## 강의 4.4 ~ 4.5 — 주 4 최종 프로젝트

| 항목 | 내용 |
|------|------|
| **목표** | 지금까지 배운 모든 것을 합쳐 매출 자동 집계기를 완성한다 |
| **소요 시간** | 약 60분 |
| **완료 기준** | 엑셀 파일을 읽어 분석하고 요약 보고서를 자동으로 저장한다 |

```python
# ── 주 4 최종 프로젝트: 매출 자동 집계기 ──────────
import openpyxl
import json
from datetime import datetime

def 엑셀_읽기(파일명):
    """엑셀 파일에서 매출 데이터를 읽어옵니다"""
    try:
        wb = openpyxl.load_workbook(파일명)
        ws = wb.active
        데이터 = []
        for 행 in ws.iter_rows(min_row=2, values_only=True):
            if 행[0]:  # 빈 행 건너뜀
                데이터.append(행)
        return 데이터
    except FileNotFoundError:
        print(f"{파일명} 파일을 찾을 수 없습니다")
        return []

def 매출_분석(데이터):
    """매출 데이터를 분석하여 요약 정보를 반환합니다"""
    메뉴별_매출 = {}
    총_매출 = 0

    for 행 in 데이터:
        월, 메뉴, 판매량, 단가, 매출 = 행
        총_매출 += 매출
        if 메뉴 not in 메뉴별_매출:
            메뉴별_매출[메뉴] = 0
        메뉴별_매출[메뉴] += 매출

    최고메뉴 = max(메뉴별_매출, key=메뉴별_매출.get)
    return {
        "총_매출": 총_매출,
        "메뉴별_매출": 메뉴별_매출,
        "최고메뉴": 최고메뉴,
        "최고메뉴_매출": 메뉴별_매출[최고메뉴]
    }

def 리포트_저장(분석결과, 출력파일명):
    """분석 결과를 JSON으로 저장합니다"""
    분석결과["생성일시"] = datetime.now().strftime("%Y-%m-%d %H:%M")
    with open(출력파일명, "w", encoding="utf-8") as f:
        json.dump(분석결과, f, ensure_ascii=False, indent=2)
    print(f"리포트 저장 완료: {출력파일명}")

# ── 실행 ─────────────────────────────────────────
데이터 = 엑셀_읽기("카페_월별_매출.xlsx")
결과 = 매출_분석(데이터)
print(f"총 매출: {결과['총_매출']:,}원")
print(f"효자 메뉴: {결과['최고메뉴']} ({결과['최고메뉴_매출']:,}원)")
리포트_저장(결과, "매출_리포트.json")
```
